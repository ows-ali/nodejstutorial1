Network request structure

HTTP is a request/response protocol

Request-which client sends to server. It has two parts .

  1. Request header-you mention HTTP verb (either POST, Get, Put etc), Auth tokens if any , cookies if any
  2. Request body-(optional)-we send form data if it is a POST/PUT/Patch request

Response-which server sends to client upon request

Respnse has two parts

  1. Header - status code (mandatory)-either 200 (201,202, all are success , rest are failed requests), 404 (not found), 403 (validation error). It can also tell Response type( json, html )
  2. Body - actual content (either json or html or text)

blocking (synchronous requests) vs non blocking (asynchronous requests)

whenever asynchrouns operation is found/met, it is removed from call stack,nd execturion proceeds

if there are more async operatiosn, then they will run as they gets completed

Whenever we use async function, it takes a callback with two arguments (err, data). So on success and failure it does one of the following (internally)

  • callback(null, data )
  • callback('file not found'(or any other error),null)

In first case, it will run else(success case). In second it will run if (err) condition

fetch('google.com',(err, data)=>{ if (err){ console.log('something went wrong') return } // else is optional, if return was wrtiten above console.log('google html is: ', data) }) fetch('googel.com',(err, data)=>{ if (err){ console.log('something went wrong') return } // else is optional, if return was wrtiten above console.log('google html is: ', data) })
